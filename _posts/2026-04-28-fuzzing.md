# Fuzzing과 Fuzzer: 취약점 분석의 핵심 자동화 기법

## 1. 퍼징(Fuzzing)이란 무엇인가?

`퍼징(Fuzz Testing)`은 소프트웨어의 보안 취약점을 발견하기 위해 사용되는 자동화된 테스트 기법이다. 프로그램에 비정상적, 예상치 못한, 혹은 무작위 데이터(Fuzz)를 입력값으로 주입하고, 그 과정에서 발생하는 크래시, 코드 단언 오류(Assertion failure), 메모리 누수 등을 모니터링한다.

보안 연구자에게 퍼징은 수동 분석으로 찾기 힘든 복잡한 버그를 대규모로 찾아낼 수 있게 해주는 필수적인 도구이다.

---

## 2. 퍼저(Fuzzer)의 분류

퍼징은 크게 대상 프로그램에 대한 이해도와 데이터 생성 방식에 따라 나뉜다.

### 2.1 대상 이해도에 따른 분류 (Knowledge-based)

* `Black-box Fuzzing`: 프로그램 내부 구조를 모르는 상태에서 입력/출력만 확인합니다. 구현이 쉽지만 효율이 떨어질 수 있다.
* `White-box Fuzzing`: 프로그램 소스 코드를 분석하여 모든 실행 경로를 체계적으로 탐색합니다. 분석 비용이 높다.
* `Grey-box Fuzzing`: 가장 대중적인 방식으로, 소스 코드의 일부 정보나 실행 흐름(Code Coverage)을 추적하며 새로운 경로를 탐색하는 입력값을 저장하고 발전시킨다. (예: AFL++)

### 2.2 입력 생성 방식에 따른 분류 (Input Generation)

* `Mutation-based` (변이 기반): 정상적인 입력값(Seed)을 바탕으로 비트를 뒤집거나 값을 수정하여 변종을 만든다.
* `Generation-based` (생성 기반): 특정 파일 형식이나 네트워크 프로토콜의 규칙(RFC 등)을 학습하여 규격에 맞는(그러나 악의적인) 데이터를 생성한다.

## 3. 대표적인 퍼징 도구 분석

미션에서 제시된 주요 도구들의 특징은 다음과 같습니다.

| 도구명 | 개발/관리 | 주요 특징 |
| :--- | :--- | :--- |
| AFL++ | Community | 기존 AFL의 개선판으로, 가장 강력한 Grey-box 퍼저입니다. 높은 성능과 다양한 변이 알고리즘을 지원합니다. |
| Jackalope | Google Project Zero | 소스 코드가 없는 블랙박스 환경(주로 Windows/macOS 바이너리)을 위해 설계된 커스텀 퍼저입니다. |
| ClusterFuzz | Google | 구글 인프라에서 수만 대의 머신을 활용해 대규모 퍼징을 수행할 수 있도록 돕는 프레임워크입니다. |

---

## 4. 퍼징 프로세스 요약

1. `대상 선정` (Targeting): 공격 표면(Attack Surface)이 넓은 라이브러리나 실행 파일을 선정한다.
2. `시드 준비` (Seed Corpus): 퍼저가 변이를 시작할 수 있는 양질의 정상 입력 파일들을 모은다.
3. `퍼징 실행` (Execution): 퍼저를 가동하여 반복적인 입력 주입을 시작한다.
4. `크래시 분석` (Analysis): 발견된 크래시가 실제 보안 취약점(CVE)으로 이어질 수 있는지 분석한다.
