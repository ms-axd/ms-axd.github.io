이번에는 `EIP-1559`를 보자.

EIP-1559는 이더리움의 가스비 구조를 바꾼 제안이다.

ERC-20, ERC-721, ERC-1155가 토큰을 어떻게 표현할지에 대한 표준이었다면,
EIP-1559는 사용자가 트랜잭션을 보낼 때 내는 수수료가 어떻게 정해지는지에 영향을 준다.

```text
ERC-20   -> 대체 가능한 토큰 표준
ERC-721  -> 대체 불가능한 토큰 표준
ERC-1155 -> 여러 종류의 토큰을 한 컨트랙트에서 관리하는 표준
EIP-1559 -> Ethereum 가스비 구조를 바꾼 제안
```

---

## EIP-1559란?

`EIP-1559`는 이더리움 트랜잭션 수수료 시장을 개선하기 위해 도입된 EIP다.

핵심은 기존의 단순 경매 방식 대신,
네트워크가 기본 수수료인 `base fee`를 정하고 사용자는 검증자에게 줄 팁인 `priority fee`를 추가하는 구조다.

그리고 중요한 점이 하나 더 있다.

`base fee`는 블록을 만든 사람에게 가지 않고 소각된다.

```text
사용자가 내는 가스비
= base fee + priority fee

base fee     -> 소각
priority fee -> 블록 제안자에게 지급
```

EIP-1559는 단순히 가스비를 싸게 만드는 제안이 아니다.

가스비를 더 예측 가능하게 만들고,
트랜잭션 수수료 구조를 더 명확하게 만드는 제안에 가깝다.

---

## 왜 등장했나

예전 이더리움의 가스비 구조는 단순했다.

사용자가 직접 `gas price`를 정해서 트랜잭션을 보냈고,
채굴자는 더 높은 수수료를 낸 트랜잭션을 우선적으로 포함했다.

```text
Alice -> gas price 30 gwei
Bob   -> gas price 50 gwei
Carol -> gas price 100 gwei
```

이 구조에서는 네트워크가 혼잡해지면 사용자가 서로 더 높은 가격을 부르는 경쟁이 생긴다.

문제는 사용자가 적당한 가격을 알기 어렵다는 점이다.

너무 낮게 잡으면 트랜잭션이 오래 대기하고,
너무 높게 잡으면 필요 이상으로 수수료를 많이 낸다.

```text
낮게 냄 -> 트랜잭션이 늦게 처리됨
높게 냄 -> 불필요하게 비싼 수수료를 냄
```

즉, 예전 구조의 문제는 단순히 비싸다는 것이 아니라
수수료 예측이 어렵고 사용자가 과하게 지불하기 쉽다는 점이었다.

---

## EIP-1559가 해결한 문제

EIP-1559가 해결하려 한 핵심 문제는 세 가지다.

### 1. 가스비 예측 문제

기존 방식에서는 사용자가 직접 적절한 `gas price`를 추측해야 했다.

EIP-1559에서는 네트워크가 `base fee`를 자동으로 계산한다.

사용자는 현재 네트워크 상황을 보고 완전히 감으로 가격을 정하는 대신,
기본 수수료 위에 어느 정도의 팁을 줄지만 결정하면 된다.

```text
예전 방식
사용자: 가스 가격을 얼마로 해야 하지?

EIP-1559 이후
네트워크: base fee는 이 정도다.
사용자: 그럼 팁을 이 정도만 더하자.
```

### 2. 과도한 수수료 지불 문제

예전 방식에서는 트랜잭션을 빨리 처리하고 싶으면 높은 가격을 제시해야 했다.

하지만 실제로는 그만큼까지 낼 필요가 없었는데도,
사용자가 불안해서 더 높은 수수료를 내는 경우가 많았다.

EIP-1559에서는 `max fee`와 `priority fee`를 나눠서 설정할 수 있다.

```text
max fee      -> 내가 낼 수 있는 최대 금액
priority fee -> 블록 제안자에게 주는 팁
```

실제 지불액이 `max fee`보다 적게 나오면 남는 금액은 사용자에게 돌아간다.

### 3. 수수료 경제 구조 문제

EIP-1559 이전에는 트랜잭션 수수료가 블록을 만든 채굴자에게 갔다.

EIP-1559 이후에는 `base fee`가 소각된다.

이 구조 때문에 네트워크 사용량이 많을수록 더 많은 ETH가 소각된다.

```text
네트워크 사용 증가
-> base fee 증가
-> 더 많은 ETH 소각
```

이것은 ETH의 공급 구조에도 영향을 준다.

---

## base fee

`base fee`는 블록에 트랜잭션을 포함하기 위해 반드시 내야 하는 기본 수수료다.

사용자가 마음대로 정하는 값이 아니라,
프로토콜이 이전 블록의 사용량을 보고 자동으로 조정한다.

블록이 많이 차면 다음 블록의 `base fee`가 올라간다.

블록이 덜 차면 다음 블록의 `base fee`가 내려간다.

```text
블록 사용량 높음 -> base fee 상승
블록 사용량 낮음 -> base fee 하락
```

이 구조 덕분에 가스비가 시장 상황을 따라가지만,
이전처럼 사용자가 완전히 감으로 가격을 찍는 방식은 줄어든다.

---

## priority fee

`priority fee`는 블록 제안자에게 주는 팁이다.

EIP-1559 이후에도 트랜잭션을 더 빨리 포함시키고 싶은 사용자는 팁을 더 줄 수 있다.

다만 예전처럼 전체 수수료를 무작정 높게 부르는 것이 아니라,
기본 수수료와 팁이 분리된다.

```text
base fee     -> 네트워크가 정함
priority fee -> 사용자가 정함
```

그래서 사용자는 네트워크 사용료와 블록 제안자에게 주는 보상을 구분해서 생각할 수 있다.

---

## max fee

`max fee`는 사용자가 이 트랜잭션에 낼 의향이 있는 최대 수수료다.

실제 지불액은 보통 다음처럼 계산된다.

```text
실제 지불액 = base fee + priority fee
```

단, 이 값은 사용자가 설정한 `max fee`를 넘을 수 없다.

예를 들어 보자.

```text
max fee: 100 gwei
base fee: 60 gwei
priority fee: 2 gwei
```

이 경우 실제 지불액은 `62 gwei`다.

사용자가 최대 `100 gwei`까지 낼 수 있다고 설정했더라도,
필요한 금액이 `62 gwei`라면 나머지는 쓰이지 않는다.

```text
100 gwei 전부 지불하는 것이 아니다.
필요한 만큼만 지불한다.
```

---

## ETH 소각

EIP-1559에서 가장 많이 언급되는 부분은 `base fee` 소각이다.

사용자가 낸 `base fee`는 블록 제안자에게 지급되지 않고 사라진다.

```text
사용자
-> base fee 지불
-> base fee 소각
```

소각된 ETH는 다시 유통되지 않는다.

그래서 EIP-1559 이후에는 이더리움 네트워크가 많이 사용될수록
ETH가 더 많이 소각되는 구조가 되었다.

다만 이것이 항상 ETH 총공급이 줄어든다는 뜻은 아니다.

새로 발행되는 ETH보다 소각되는 ETH가 많을 때는 공급이 줄 수 있고,
반대로 소각량이 적으면 공급은 늘 수 있다.

---

## EIP-1559가 가스비를 낮췄나

여기서 헷갈리기 쉬운 점이 있다.

EIP-1559는 가스비를 무조건 싸게 만드는 제안이 아니다.

네트워크가 혼잡하면 가스비는 여전히 비싸다.

EIP-1559가 바꾼 것은 가스비의 절대 가격보다는
수수료를 정하고 지불하는 방식이다.

```text
EIP-1559 이전
사용자가 gas price를 직접 찍음

EIP-1559 이후
프로토콜이 base fee를 제시하고
사용자는 priority fee와 max fee를 설정함
```

즉, 목적은 수수료를 더 예측 가능하게 만들고
불필요한 과지불을 줄이는 것이다.

---

## 간단한 예시

NFT를 하나 민팅한다고 해보자.

EIP-1559 이전에는 사용자가 이런 고민을 해야 했다.

```text
지금 gas price를 30 gwei로 할까?
50 gwei로 할까?
100 gwei로 해야 빨리 들어갈까?
```

네트워크가 갑자기 붐비면 낮게 잡은 트랜잭션은 오래 대기할 수 있다.

반대로 너무 높게 잡으면 실제로 필요한 것보다 많이 낸다.

EIP-1559 이후에는 구조가 바뀐다.

```text
현재 base fee: 40 gwei
priority fee: 2 gwei
max fee: 70 gwei
```

사용자는 현재 기본 수수료를 보고,
얼마나 빨리 처리하고 싶은지에 따라 팁을 조절한다.

네트워크가 조금 변해도 `max fee` 안에서 처리될 수 있고,
불필요하게 남은 금액은 사용되지 않는다.

---

## 왜 중요한가

EIP-1559가 중요한 이유는 이더리움의 사용자 경험과 경제 구조를 동시에 건드렸기 때문이다.

첫째, 사용자가 트랜잭션 수수료를 이해하기 쉬워졌다.

둘째, 지갑이 수수료를 추천하기 쉬워졌다.

셋째, `base fee` 소각으로 ETH의 공급 구조가 바뀌었다.

넷째, 이더리움 수수료 시장이 더 명확한 규칙을 가지게 되었다.

```text
수수료 예측 개선
과지불 감소
base fee 소각
ETH 경제 구조 변화
```

EIP-1559는 단순한 가스비 패치가 아니라,
이더리움이 수수료를 다루는 방식을 바꾼 핵심 EIP다.

---

## 한계

EIP-1559가 모든 문제를 해결한 것은 아니다.

네트워크가 혼잡하면 가스비는 여전히 높다.

`base fee`는 수요가 많으면 올라갈 수밖에 없다.

그리고 사용자가 빠른 처리를 원하면 여전히 `priority fee`를 높게 줄 수 있다.

```text
혼잡한 네트워크
-> base fee 상승
-> 총 수수료 증가
```

그래서 EIP-1559 이후에도 확장성 문제는 남아 있다.

이 문제는 이후 `EIP-4844` 같은 제안과 L2 확장 솔루션으로 이어진다.

---

## 정리

EIP-1559는 이더리움의 가스비 구조를 바꾼 제안이다.

핵심은 세 가지다.

```text
base fee     -> 네트워크가 정하는 기본 수수료, 소각됨
priority fee -> 블록 제안자에게 주는 팁
max fee      -> 사용자가 낼 수 있는 최대 수수료
```

EIP-1559는 가스비를 무조건 싸게 만드는 제안이 아니다.

대신 수수료를 더 예측 가능하게 만들고,
사용자의 과지불을 줄이고,
ETH 소각 구조를 도입했다.

그래서 EIP-1559는 이더리움의 수수료 시장과 ETH 경제 구조를 이해할 때 반드시 봐야 하는 핵심 EIP다.
