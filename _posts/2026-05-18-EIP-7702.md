EIP-7702는 EOA가 스마트 컨트랙트 계정처럼 동작할 수 있게 만드는 EIP다.

핵심은 단순하다.

```text
EOA에 직접 코드를 배포하는 것이 아니라
특정 컨트랙트 코드로 위임한다.
```

즉, 기존 지갑 주소를 유지하면서 스마트 계정 기능을 붙이는 방식이다.

---

## 기존 EOA의 한계

EOA는 Ethereum에서 가장 기본적인 지갑 형태다.

개인키가 있고,
그 개인키로 트랜잭션에 서명한다.

구조가 단순해서 이해하기 쉽다.

하지만 기능도 제한적이다.

```text
트랜잭션 묶기 어려움
가스비를 직접 ETH로 내야 함
권한을 세분화하기 어려움
세션 키를 기본 지원하지 않음
복구 로직을 계정 자체에 넣기 어려움
```

예를 들어 DEX에서 토큰을 교환하려면 보통 두 번의 트랜잭션이 필요하다.

```text
1. approve
2. swap
```

사용자 입장에서는 번거롭다.

스마트 계정이라면 이런 작업을 하나로 묶을 수 있다.

---

## 스마트 컨트랙트 지갑의 장점

스마트 컨트랙트 지갑은 코드로 동작한다.

그래서 계정 안에 다양한 규칙을 넣을 수 있다.

```text
멀티시그
소셜 복구
가스 대납
트랜잭션 묶기
세션 키
일일 지출 한도
특정 dApp 권한만 허용
```

문제는 기존 사용자가 이미 EOA를 쓰고 있다는 점이다.

자산, NFT, 활동 기록, 주소 기반 신뢰가 모두 기존 EOA에 묶여 있다.

새 스마트 컨트랙트 지갑으로 옮기는 것은 사용자에게 부담이다.

EIP-7702는 이 문제를 직접 겨냥한다.

```text
기존 EOA 주소는 유지한다.
대신 그 EOA가 특정 코드처럼 동작하게 만든다.
```

---

## Set Code Transaction

EIP-7702는 새로운 트랜잭션 타입을 추가한다.

이 트랜잭션을 `set code transaction`이라고 부른다.

공식 문서 기준 트랜잭션 타입은 다음 값이다.

```text
SET_CODE_TX_TYPE = 0x04
```

이 트랜잭션에는 `authorization_list`가 들어간다.

```text
authorization_list = [
  [chain_id, address, nonce, y_parity, r, s],
  ...
]
```

여기서 핵심은 `address`다.

EOA가 어떤 컨트랙트 코드로 위임할지 가리킨다.

---

## Authorization Tuple

`authorization tuple`은 EOA가 코드 위임을 승인했다는 증거다.

구성은 다음과 같다.

```text
chain_id
address
nonce
y_parity
r
s
```

EOA는 이 정보를 서명한다.

클라이언트는 서명을 검증해서 어떤 EOA가 코드 위임을 허용했는지 확인한다.

```text
authorization tuple
-> 서명 검증
-> authority 주소 복구
-> authority의 코드 위임 설정
```

여기서 `authority`는 코드 위임을 승인한 EOA다.

---

## Delegation Indicator

EIP-7702는 EOA의 코드에 실제 로직 전체를 넣지 않는다.

대신 위임 표시자를 기록한다.

```text
0xef0100 || address
```

이 값을 `delegation indicator`로 볼 수 있다.

의미는 이렇다.

```text
이 계정이 호출되면
address에 있는 코드를 가져와
이 계정의 컨텍스트에서 실행한다.
```

즉, 로직은 별도 컨트랙트에 있고,
EOA는 그 로직을 가리킨다.

---

## 어떻게 실행되는가

EIP-7702가 적용된 EOA는 일반 EOA처럼 주소를 유지한다.

하지만 호출될 때는 위임된 코드가 실행된다.

```text
사용자 EOA
-> delegation indicator 보유
-> 특정 컨트랙트 주소를 가리킴
-> 호출 시 해당 컨트랙트 코드를 실행
```

중요한 점은 실행 컨텍스트다.

코드는 위임 대상 컨트랙트에서 가져오지만,
실행은 EOA의 컨텍스트에서 일어난다.

```text
코드 위치
-> delegate address

상태와 주소
-> authority EOA
```

이 구조 덕분에 기존 EOA 주소를 유지하면서 스마트 계정 기능을 사용할 수 있다.

---

## 가능한 기능

EIP-7702가 목표로 하는 대표 기능은 세 가지다.

```text
Batching
Sponsorship
Privilege de-escalation
```

하나씩 보면 이렇다.

---

## Batching

Batching은 여러 작업을 하나의 트랜잭션으로 묶는 것이다.

예를 들어 DEX 사용자는 보통 두 단계를 거친다.

```text
1. approve
2. swap
```

EIP-7702를 사용하면 지갑 로직이 이 작업을 한 번에 처리할 수 있다.

```text
approve + swap
-> 하나의 흐름으로 실행
```

사용자는 트랜잭션을 여러 번 승인하지 않아도 된다.

---

## Sponsorship

Sponsorship은 다른 주체가 가스비를 대신 내는 구조다.

예를 들어 dApp이 신규 사용자의 가스비를 대신 낼 수 있다.

```text
사용자
-> 작업 승인

스폰서
-> 트랜잭션 비용 지불
```

이 구조는 온보딩에 중요하다.

처음 지갑을 만든 사용자가 ETH를 갖고 있지 않아도 서비스를 사용할 수 있기 때문이다.

---

## Privilege De-escalation

Privilege de-escalation은 권한을 줄여서 위임하는 것이다.

개인키 전체 권한을 넘기는 것이 아니라,
제한된 권한만 부여한다.

예시는 이렇다.

```text
특정 dApp에서만 사용 가능
하루에 일정 금액까지만 사용 가능
ETH는 못 쓰고 특정 ERC-20만 사용 가능
일정 시간 뒤 만료
```

이런 구조는 세션 키와 잘 맞는다.

사용자는 매번 지갑 팝업을 승인하지 않아도 되고,
권한은 제한된 범위 안에서만 동작한다.

---

## ERC-4337과의 차이

ERC-4337과 EIP-7702는 둘 다 지갑 UX 개선을 목표로 한다.

하지만 접근 방식이 다르다.

```text
ERC-4337
-> UserOperation, Bundler, EntryPoint 기반
-> Smart Account 중심
-> 프로토콜 변경 없이 인프라 계층에서 구현

EIP-7702
-> set code transaction 기반
-> 기존 EOA를 확장
-> 새로운 트랜잭션 타입 사용
```

ERC-4337은 스마트 컨트랙트 계정을 기본 계정처럼 쓰게 한다.

EIP-7702는 기존 EOA가 스마트 계정 기능을 사용할 수 있게 한다.

---

## EIP-1271과의 연결

EIP-1271은 스마트 컨트랙트 지갑의 서명 검증 표준이다.

EIP-7702는 EOA의 실행 방식을 스마트 계정 쪽으로 확장한다.

둘은 직접 같은 문제를 다루지는 않지만,
큰 흐름은 연결된다.

```text
EIP-1271
-> 계정 로직이 서명을 검증

EIP-7702
-> EOA가 계정 로직을 실행할 수 있게 함
```

지갑이 단순 개인키 계정에서 코드 기반 계정으로 이동할수록,
서명 검증과 권한 관리도 코드 중심으로 바뀐다.

---

## 주의할 점

EIP-7702는 강력하지만 위험도 크다.

EOA가 어떤 코드로 위임되는지가 매우 중요하다.

잘못된 위임 코드는 계정 자산을 위험하게 만들 수 있다.

```text
악성 delegate code
잘못된 권한 관리
초기화 과정의 취약점
스토리지 충돌
프론트러닝
과도한 권한의 세션 키
```

특히 사용자는 자신이 어떤 코드에 계정을 위임하는지 이해하기 어렵다.

그래서 지갑 UI와 보안 검증이 중요하다.

---

## 정리

EIP-7702는 기존 EOA가 스마트 계정처럼 동작할 수 있게 하는 EIP다.

새로운 `set code transaction`을 통해 EOA에 delegation indicator를 설정한다.

EOA는 특정 컨트랙트 코드를 가리키고,
호출 시 그 코드를 자기 컨텍스트에서 실행한다.

```text
기존 EOA
-> 코드 위임 설정
-> 스마트 계정 기능 사용
```

이 방식으로 batching, sponsorship, 제한된 권한 위임 같은 기능을 기존 지갑 주소 위에 올릴 수 있다.
