이번에는 `EIP-4844`를 보자.

EIP-4844는 이더리움 L2 수수료를 낮추기 위해 도입된 핵심 EIP다.

정식 이름은 `Proto-Danksharding`이다.

이름은 어렵지만 핵심은 단순하다.

```text
L2가 데이터를 Ethereum에 올릴 때
기존 calldata보다 더 싸게 올릴 수 있는 전용 공간을 만든다.
```

그 전용 데이터 공간이 `blob`이다.

---

## EIP-4844란?

`EIP-4844`는 이더리움에 `blob-carrying transaction`을 추가한 제안이다.

기존 트랜잭션은 실행에 필요한 데이터와 일반 데이터를 모두 `calldata`에 담았다.

하지만 L2 rollup은 Ethereum 위에서 직접 모든 연산을 실행하려는 것이 아니다.

L2에서 트랜잭션을 처리하고,
그 결과를 검증할 수 있는 데이터를 Ethereum에 게시한다.

```text
L2에서 트랜잭션 처리
-> 결과와 검증용 데이터 생성
-> Ethereum에 데이터 게시
```

문제는 이 데이터를 `calldata`로 올리면 비용이 비싸다는 점이다.

EIP-4844는 이 문제를 해결하기 위해 `blob`이라는 별도의 데이터 공간을 도입했다.

---

## 왜 등장했나

Ethereum L1은 비싸다.

모든 사용자가 L1에서 직접 트랜잭션을 실행하면 네트워크가 금방 혼잡해진다.

그래서 rollup 같은 L2가 등장했다.

```text
L1 -> 보안과 최종 정산
L2 -> 많은 트랜잭션 처리
```

L2는 많은 트랜잭션을 묶어서 처리한 뒤,
필요한 데이터를 L1에 올린다.

이때 L1에 데이터를 올리는 비용이 L2 수수료의 큰 부분을 차지한다.

```text
L2 수수료
= L2 실행 비용 + L1 데이터 게시 비용
```

L2에서 실행 자체는 싸더라도,
L1에 데이터를 올리는 비용이 비싸면 사용자가 체감하는 수수료도 높아진다.

EIP-4844는 바로 이 `L1 데이터 게시 비용`을 낮추기 위해 등장했다.

---

## calldata의 문제

기존에는 L2가 데이터를 Ethereum에 올릴 때 주로 `calldata`를 사용했다.

`calldata`는 스마트 컨트랙트 호출에 포함되는 데이터다.

문제는 calldata가 Ethereum 상태 실행과 연결된 일반 트랜잭션 데이터라는 점이다.

L2 입장에서는 이 데이터가 영구적으로 비싼 공간에 남아 있을 필요가 없다.

필요한 것은 일정 기간 동안 검증할 수 있는 데이터다.

```text
L2가 필요한 것
-> 일정 기간 동안 데이터 가용성 보장

꼭 필요하지 않은 것
-> 모든 노드가 영구적으로 비싼 방식으로 저장
```

그래서 EIP-4844는 L2 데이터에 맞는 별도 공간을 만든다.

---

## blob

`blob`은 대량의 데이터를 담기 위한 공간이다.

스마트 컨트랙트가 blob 데이터를 직접 읽지는 못한다.

대신 blob에 대한 commitment를 확인할 수 있다.

```text
blob
-> 큰 데이터 덩어리

commitment
-> 그 blob이 어떤 데이터인지 증명하기 위한 값
```

즉, blob은 실행용 데이터가 아니라 데이터 가용성을 위한 공간에 가깝다.

L2는 트랜잭션 데이터를 blob에 담아 Ethereum에 게시한다.

Ethereum은 이 blob 데이터가 일정 기간 사용 가능하도록 보장한다.

---

## blob-carrying transaction

EIP-4844는 새로운 형태의 트랜잭션을 추가한다.

이를 `blob-carrying transaction`이라고 한다.

이 트랜잭션은 일반 트랜잭션 데이터와 함께 blob을 가진다.

```text
일반 트랜잭션
-> 실행에 필요한 데이터 포함

blob-carrying transaction
-> 실행 데이터 + blob 데이터 포함
```

하지만 blob 데이터는 EVM에서 직접 실행되지 않는다.

이 점이 중요하다.

blob은 컨트랙트 로직을 실행하기 위한 데이터가 아니라,
rollup이 자신의 데이터를 L1에 게시하기 위한 데이터다.

---

## blob fee

EIP-4844는 blob을 위한 별도 수수료 시장도 만든다.

기존 가스비와 blob 수수료는 분리된다.

```text
일반 gas fee -> 일반 트랜잭션 실행 비용
blob fee     -> blob 데이터 게시 비용
```

이 구조는 EIP-1559와 비슷하게 동작한다.

blob 사용량이 많으면 blob fee가 올라가고,
blob 사용량이 적으면 blob fee가 내려간다.

```text
blob 수요 증가 -> blob fee 상승
blob 수요 감소 -> blob fee 하락
```

일반 트랜잭션 실행 수요와 L2 데이터 게시 수요를 분리해서 다룰 수 있다는 점이 핵심이다.

---

## L2 수수료가 낮아지는 이유

L2 수수료에서 큰 비중을 차지하는 것은 L1 데이터 게시 비용이다.

EIP-4844는 L2가 calldata 대신 blob을 사용할 수 있게 만든다.

blob은 L2 데이터 게시 목적에 맞게 설계되었기 때문에,
calldata보다 비용이 낮다.

```text
EIP-4844 이전
L2 데이터 -> calldata 사용 -> 비쌈

EIP-4844 이후
L2 데이터 -> blob 사용 -> 더 저렴
```

그래서 rollup 사용자의 수수료가 낮아질 수 있다.

특히 Optimism, Arbitrum, Base, zkSync 같은 L2가 영향을 받는다.

---

## Danksharding과의 관계

EIP-4844의 별칭은 `Proto-Danksharding`이다.

`proto`는 완성형이 아니라 중간 단계라는 뜻에 가깝다.

즉, EIP-4844는 완전한 Danksharding을 바로 구현한 것이 아니다.

나중에 더 큰 확장성 구조로 가기 전에,
blob 트랜잭션과 blob 수수료 시장을 먼저 도입한 단계다.

```text
EIP-4844
-> blob 도입
-> L2 데이터 게시 비용 감소
-> 완전한 Danksharding으로 가기 위한 중간 단계
```

그래서 EIP-4844는 Ethereum 확장성 로드맵에서 중요한 위치를 가진다.

---

## EIP-1559와 비교

EIP-1559는 일반 트랜잭션 수수료 시장을 바꿨다.

EIP-4844는 blob 데이터 수수료 시장을 추가했다.

둘 다 수요에 따라 기본 수수료가 조정되는 구조를 가진다.

```text
EIP-1559
-> 일반 gas base fee

EIP-4844
-> blob base fee
```

하지만 목적은 다르다.

EIP-1559는 일반 트랜잭션 수수료를 더 예측 가능하게 만드는 데 초점이 있다.

EIP-4844는 L2가 데이터를 더 싸게 게시할 수 있도록 하는 데 초점이 있다.

---

## 간단한 예시

L2에서 사용자가 NFT를 민팅한다고 해보자.

L2는 여러 사용자의 트랜잭션을 묶어서 처리한다.

그리고 그 결과를 검증할 수 있는 데이터를 Ethereum에 올린다.

EIP-4844 이전에는 이 데이터를 calldata에 올렸다.

```text
사용자 트랜잭션 여러 개
-> L2에서 처리
-> 데이터 calldata로 L1에 게시
-> 비용 높음
```

EIP-4844 이후에는 blob을 사용할 수 있다.

```text
사용자 트랜잭션 여러 개
-> L2에서 처리
-> 데이터 blob으로 L1에 게시
-> 비용 감소
```

사용자 입장에서는 L2 수수료가 낮아지는 효과를 볼 수 있다.

---

## 한계

EIP-4844가 모든 확장성 문제를 끝낸 것은 아니다.

L2 수수료를 낮추는 데 큰 도움이 되지만,
Ethereum L1 자체의 실행 처리량을 크게 늘리는 제안은 아니다.

또 blob 공간도 무한하지 않다.

많은 L2가 동시에 blob을 사용하면 blob fee는 올라갈 수 있다.

```text
blob 공간 수요 증가
-> blob fee 상승
-> L2 수수료도 다시 상승 가능
```

즉, EIP-4844는 확장성의 끝이 아니라 중요한 중간 단계다.

---

## 왜 중요한가

EIP-4844가 중요한 이유는 Ethereum의 확장 방향이 L2 중심이라는 점과 연결된다.

Ethereum은 L1에서 모든 것을 직접 처리하기보다,
L2가 많은 트랜잭션을 처리하고 L1은 보안과 데이터 가용성을 제공하는 방향으로 발전하고 있다.

```text
L2 -> 실행 확장
L1 -> 보안, 정산, 데이터 가용성
```

EIP-4844는 이 구조에서 L2가 L1을 더 싸게 사용할 수 있게 만든다.

그래서 L2 사용자 경험과 Ethereum 확장성에 직접적인 영향을 준다.

---

## 정리

EIP-4844는 L2 수수료를 낮추기 위해 `blob`을 도입한 EIP다.

핵심은 세 가지다.

```text
blob
-> L2 데이터 게시를 위한 전용 데이터 공간

blob-carrying transaction
-> blob을 포함할 수 있는 새로운 트랜잭션

blob fee
-> 일반 gas fee와 분리된 blob 전용 수수료
```

EIP-4844는 완전한 Danksharding은 아니다.

하지만 rollup 중심 확장 구조로 가기 위한 중요한 단계다.

그래서 EIP-1559가 가스비 구조를 이해하는 핵심 EIP라면,
EIP-4844는 L2 수수료와 Ethereum 확장성을 이해하는 핵심 EIP다.

```text
각 EIP의 한게를 보완한 다른 EIP가 있다보니 이어져서 알고 싶던 
지갑 동작 방식, 스마트 컨트렉트 설계 방식은 아직 보지 못했지만 추후에 공부하고 글 작성해야 겠다.
```
