### rot13

알파벳 문자를 `13` 글자 뒤의 문자로 바꾸는 단순 치환 암호

```bash
cat data.txt | tr 'A-Za-z' 'N-ZA-Mn-za-m'

tr : 표준 입력으로부터 문자를 읽어 치환하는 명령어
# 'A-Za-z' : 변환할 대상 범위 지정,  모든 대소문자를 의미
## A부터 M까지(1~13번째)는 N부터 Z로 바뀐다
###  N부터 Z까지(14~26번째)는 다시 A부터 M으로 바뀐다.
```

## Bandit12

![alt text](/assets/images/bandit/image-28.png)

이게 젤 귀찮다 ... 

여러번 압축된 파일 압축 해제 하는 문제다

일단 문제 설명 따라 

```bash
mkdir /tmp/flag
cp data.txt /tmp/flag
cd /tmp/flag
```

`data.txt` 파일은 `hexdump` 로 추출된 텍스트 데이터기 때문에 
`xxd` 를 이용해보자

```bash
xxd -r data.txt /tmp/flag
```

하면 `data` 파일이 생기는데 `file`로 타입을 확인하고 파일명 뒤에 타입에 맞는 확장자를 붙여야 압축을 풀 수 있다.

```bash
mv 원래_이름 바꿀_이름 # 예시 : mv test.txt flag.gz
gzip -d 파일이름
bz2 -d 파일이름
tar -xvf 파일이름
```

그렇게 노가다를 하면 `ASCII text` 타입 파일이 생긴다...

이 문제는 `SSH` 개인 키를 사용해 `bandit14`에 로그인하는 문제다

`ssh -i file_name bandit14@~~~~`머시기머시기 

여기서 `-i file` 은 비밀번호 대신 이 파일을 증거로 증거로 제출할게요~ 이다.

![alt text](/assets/images/bandit/image-31.png)

근데 안되노

이게 그 뭐지 대충 계정에 `ssh`권한이 없어서 그렇다, 설정이 그렇게 되있다라고 했던거 같은데

그리고 `ssh` 개인키로 들어갈라면 파일 권한을 소유자만 읽고 쓸 수 있게 변경해야 한다 라고 하더라고요

쨋든 그래서 전 `scp` 로 파일을 가져왔습니다. 

```bash
scp -P 2220 bandit13@bandit.labs.overthewire.org:~/sshkey.private ~/sshkey.private
# 이렇게 가져오고

chmod 600 ~/sshkey.private 
# 권한 변경해주고 
```
하고 다시 접속해보면 

`good !`

문제 설명대로 

```bash
cat /etc/bandit_pass/bandit14

```

## Bandit14

![alt text](/assets/images/bandit/image-33.png)

`bandit14`의 `pw`를 `30000` 포트에 전송하라네요

```bash
telnet localhost 30000 #하고 pw 전송
    or
nc localhost 30000 # |  옵션 붙여서 해도 되네
```

### telnet vs nc

요 둘의 차이점은 

| `telnet` | `nc` | 
| :----: | :----: | 
| 텍스트 기반 원격 터미널 접속 | 네트워크 분석 및 접속 | 
| 주로 TCP만 지원 | TCP UDP 모두 지원 |
| 데이터 전송 위주 | 서버 모드(-l), 포트스캔 등 다양함 |

## Bandit15

![alt text](/assets/images/bandit/image-35.png)

`openssl` 이용해서 `30001`  pw 전송

```bash
openssl s_client -connect localhost:30001
```

### openssl s_client 가 뭐냐 

`SSL/TLS 암호화` 통신을 가능하게 해주는 강력한 도구

`nc` 나 `telnet`이 평문으로 데이터를 주고받는다면 `openssl s_client`는 데이터를 암호화해서 안전하게 주고 받는다.
